## 5. 核心功能验收

### 5.1 严格编号引用

v0.1.2 采用严格编号引用格式：

```markdown
AI 搜索正在快速发展。[1]

## Sources

[1] https://example.com/source-a
```

验证规则：

- 必须存在 `## Sources`
- 正文必须至少出现一个 `[n]`
- 正文引用的编号必须在 Sources 中定义
- Sources 中每个编号必须被正文引用
- Sources URL 必须来自 Tavily 搜索结果
- 正文不允许裸 URL

验收结果：通过。

### 5.2 自动重写一次

当第一次报告引用校验失败时，系统会：

1. 记录第一次失败原因
2. 构造 rewrite prompt
3. 重新调用 LLM 生成一次完整报告
4. 再次执行 citation validation
5. 根据第二次结果保存成功报告或失败报告

在线验收中，第一个问题触发了 retry：

```text
question: AI 搜索引擎的发展趋势
rewrite_attempted: True
validation_attempts: 2
attempt 1 failure: unused_sources
final result: success
```

验收结果：通过。

### 5.3 Review 只审核最终报告

LangGraph 主结构保持：

```text
plan_research → search_web → synthesize_notes → write_report → review_report → save_report
```

retry 被封装在 `write_report` 节点内部，`review_report` 只接收最终 report。

验收结果：通过。

### 5.4 完整失败报告

如果 retry 后仍失败，失败报告会包含：

- 第一次失败原因
- 第二次失败原因
- reason
- body citations
- source citations
- undefined citations
- unused sources
- invalid source URLs
- bare body URLs
- allowed URLs
- 用户建议

验收结果：通过。

### 5.5 Verbose retry metadata

`--verbose` 会展示：

```text
Report validation:
- rewrite_attempted: True/False
- validation_attempts: 1/2
- final_status: success/failed_validation
- attempt 1: <reason>
- attempt 2: <reason>
```

验收结果：通过。

## 6. 已知限制

v0.1.2 仍然不是最终形态的 Deep Research Agent。

已知限制：

1. 每个子问题仍只有一个 search query。
2. 不做搜索结果 rerank。
3. 不做来源质量评分。
4. 不做 review 失败后的补充搜索。
5. 不保存 trace JSON。
6. 不支持 PDF/DOCX 导出。
7. 不支持多 Agent 协作。
8. 在线成功率已显著提升，但仍受 LLM 输出随机性影响。
